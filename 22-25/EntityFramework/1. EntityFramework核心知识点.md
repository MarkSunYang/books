一、核心概念与基础
1. 什么是 Entity Framework？

EF 是一个由微软提供的开源 ORM 框架。它让开发者可以使用 .NET 对象来与数据库进行交互，无需编写大部分的数据访问代码。它充当了应用程序中的领域模型（类）和数据库表之间的桥梁。

关键点：ORM - 对象关系映射器。

2. EF 的主要组件有哪些？

DbContext：这是与数据库交互的主要类。它代表一个工作单元和仓库模式的组合。

工作单元：跟踪所有已加载的实体，并在一次调用中将所有更改持久化到数据库。
仓库模式：DbSet<T> 属性充当实体集合的仓库。
DbSet<T>：表示上下文中的特定实体集合（通常对应一张数据库表）。
Entity：普通的 .NET 类（POCO），映射到数据库中的一张表。


### Student 实体类
```
public class Student
{
    public int StudentId { get; set; }
    public string Name { get; set; }
}
```
### 在DbContext中，我们会有这样一个DbSet：
```
public class SchoolContext : DbContext
{
    public DbSet<Student> Students { get; set; }
}
```
Entity（实体） 是数据的模型，它是一个类，其每个实例对象代表数据库中的一行记录。它定义了数据的结构和关系。
DbSet<T> 是数据访问的仓库，它是 DbContext 中的一个属性，代表数据库中的整张表。我们通过它来对 T 类型的实体集合进行查询、添加、删除等操作。

3. EF 有哪三种开发模式？

Database First：从已有的数据库生成实体模型（.edmx 文件）和上下文代码。适用于老项目或数据库已存在的情况。
Model First：在 Visual Studio 的设计器中先创建实体模型（.edmx），然后根据模型生成数据库和代码。现在较少使用。
Code First：目前的主流和推荐方式。你先编写实体类（Code）和 DbContext，然后 EF 根据这些代码来创建和管理数据库。它强调通过代码来定义模型，非常灵活。

4. 什么是迁移？它的作用是什么？

迁移是 Code First 模式中，使数据库模式与应用程序的实体模型保持同步的一种方式。
作用：当你的实体类发生变化时（如添加新属性、新类），你可以通过 Add-Migration <Name> 创建一个迁移文件，该文件包含了升级（Up）和降级（Down）数据库的 C# 代码。然后使用 Update-Database 命令将这些更改应用到数据库。这实现了数据库的版本控制。
